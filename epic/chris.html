All of his friends are in relationships, so he is sick of being the 3rd or 5th wheel when they all go out. He wants to eventually get married and start a family.

Use Case

Title:Receives an email notification

Description: Chris receives an email notification from dating site URL, logs into site to check status updates

Name of the User and Their Role: Chris Smith--potential love match

Usage Preconditions: Must have internet capable device, must be logged in

Usage Post-conditions: Server sends a message to his most recent match and saves his most recent Personality updates

Interaction Flow:

  • Chris receives an email notification from URL of dating website
  • After work, at home, Chris opens email account in Chrome to look at email
  • Chris enters URL in Chrome to dating site
  • Server receives the request and sends him to the dating site
  • Chris clicks his like list
  • Server responds sending him to the list
  • Chris looks for the woman who just messaged him
  • Chris finds the illuminated heart and clicks link to her profile
  • Chris messages her back in message box
  • Server receives request and sends an email notification to woman's email account
  • Chris decides to update his Personality section, so clicks link to editor
  • Server receives the request and sends him to the editor
  • Chris updates his Personality section and clicks finish
  • Server receives request and finishes update, making changes

Frequency of Use: Daily
