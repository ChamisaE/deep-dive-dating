  • Age: 30
  • Ethnicity: Mixed- Half hispanic and half caucasian

User Story

Adrien feels like her life is fulfilled in all aspects. Her career is on point, her dog is a great companion and she is ready for a boo thang. She's fed up with dating sites promote hook ups. She wants to have a real connection and fall in love.

Use Case

Title: Looking for a boyfriend

Description: Adrien gets on the site to find love

Name of the User and Their Role: Adrien Baca-Garcia is the potential love match

Usage Preconditions: Internet capable device, must be logged into account

Usage Post-conditions: Server saves two instances to the Match entity and sends an email notification to Adrien's love interest.

Interaction Flow:

  • Adrien enters blamedandatedan.online on her laptop using Chrome browser.
  • Server receives requested site and, since she is logged in, returns her profile page.
  • Adrien looks to the menu on the left and clicks the link for the Browse page.
  • Server receives the request and returns the page with an array of Adrien's randomized suggested matches.
  • She scrolls through some of the suggestions and sees a really cute guy. Adrien clicks the link for his profile page.
  • Server receives the request and returns the users profile page.
  • Adrien begins to read the profile and is impressed. He has Dan rating of B and she wants to know more so she clicks the "like button".
  • Server receives the request and saves the "like" to the Match entity. Server checks if the inverse of the most recent Match row is true. If true the hearts are made red and hidden content is revealed, if false nothing happens.
  • Nothing happens. Dissatisfied with this, Adrien wants to keep searching her suggestions, so she clicks on the menu link for her browse section.
  • Server receives request and returns the page with an array of Adrien's randomized suggested matches.
  • Adrien scrolls through the suggestions once again noticing that her recent "like" is not visible on this page anymore. However she does see a profile picture of a handsome man cooking. She immediately clicks on his link.
  • Server receives request and returns this new profile page.
  • Adrien begins reading and is amazed. Not only does this man know how to cook, he can speak three languages, he is also a Tech Recruiter, but his Dan Rating is an A+! Adrien expediently clicks the "like button".
  • Server receives the request and saves the "like" to the Match entity. Server checks if the inverse of the most recent Match row is true. If true the hearts are made red and hidden content is revealed, if false nothing happens.
  • Adrien sees that a number of things have changed on the man's profile including the heart at the top of the page is now red and that there is message box she can use, indicating that it's a match. Adrien composes a message to send to the man and clicks submit.
  • Server receives request and sends an email notification to Adrien's match's email account.
  • Adrien is now content with her search effort today, she closes the window but doesn't log out of blamedandatedan.com

Frequency of Use: Three to four times a week.
